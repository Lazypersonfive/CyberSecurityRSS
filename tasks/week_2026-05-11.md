# Week 2026-05-11 Execution Plan

## Main Goal
把 AIHOT 式质量架构从“计划”推进到“可观测、可回测、可调参”的实际流水线。优先做能直接提升日报质量和调试效率的部分，不先做对外 API/RSS/Skill。

## Priority Tasks
1. **Source Audit + Registry v1**
   - 每日自动生成 `reports/source_audit.md`。
   - 统计各板块 T1/T1.5/T2/unknown/Google News 分布。
   - 列出入选条目中的未登记源，作为次日人工审阅清单。
   - 目标：入选条目 `unknown_items=0`，新增源必须显式登记。

2. **Security Source Quality Pass**
   - 基于 source audit 清理 security 的低质/泛资讯/历史归档源。
   - 把中文安全源分为中文官方、中文安全、中文公众号、社区。
   - 目标：security 每日 15 条中至少 6 条中文，并且不是低质中文硬凑。

3. **Deterministic Event Clustering v1**
   - 先做 CVE、规范化 URL、标题实体规则聚类。
   - LLM dedupe 继续作为兜底。
   - 目标：同一 CVE/同一发布不重复占位。

4. **Offline Eval v1**
   - 用最近 7-14 天 docs/digest/output 回放 source policy 和 clustering。
   - 输出旧/新策略差异、中文比例、Google/X/官网占比。
   - 目标：调权重前先看回测结果。

5. **Multidimensional Scoring Preparation**
   - 定义每个板块 `score_dimensions` schema 和 final_score 权重配置。
   - 暂不一次性切生产，先通过 mock/dry-run 验证 schema。

6. **AIHOT Compare Tool**
   - 拉 AIHOT selected items 作为 AI 板块外部 benchmark。
   - 不作为生产主信源。

## This Week Acceptance Criteria
- `reports/source_audit.md` 在 daily workflow 中自动生成并提交。
- 过去 7 天 feed 的 selected unknown 源可见、可追踪。
- 当前最新日报 selected unknown 源降到 0。
- 测试、ruff、GitHub Pages 部署保持绿色。
