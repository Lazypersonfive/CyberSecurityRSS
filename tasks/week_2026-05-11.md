## Priority Tasks
1. **Source Audit + Registry v1**
   - 每日自动生成 `reports/source_audit.md`。
   - 统计各板块 T1/T1.5/T2/unknown/Google News 分布。
   - 列出入选条目中的未登记源，作为次日人工审阅清单。

   - Status 2026-05-11: done. `reports/source_audit.md` 已接入 daily，本周 selected unknown 已归零。

2. **Security Source Quality Pass**
   - 基于 source audit 清理 security 的低质/泛资讯/历史归档源。
   - 把中文安全源分为中文官方、中文安全、中文公众号、社区。
   - 目标：security 每日 15 条中至少 6 条中文，并且不是低质中文硬凑。

3. **Deterministic Event Clustering v1**
   - 先做 CVE、规范化 URL、标题实体规则聚类。
   - LLM dedupe 继续作为兜底。
   - 目标：同一 CVE/同一发布不重复占位。

4. **Offline Eval v1**
   - 用最近 7-14 天 docs/digest/output 回放 source policy 和 clustering。
   - 输出旧/新策略差异、中文比例、Google/X/官网占比。
   - 目标：调权重前先看回测结果。
   - Status 2026-05-11: done. `reports/offline_eval.md` 已接入 daily，并修正中文源统计不能使用生成后的中文摘要。

5. **Multidimensional Scoring Preparation**
   - 定义每个板块 `score_dimensions` schema 和 final_score 权重配置。
   - 暂不一次性切生产，先通过 mock/dry-run 验证 schema。

6. **AIHOT Compare Tool**
   - 拉 AIHOT selected items 作为 AI 板块外部 benchmark。
   - 不作为生产主信源。
   - Status 2026-05-11: done as manual tool. `aihot_compare.py` 输出 `reports/aihot_compare.md`，不接入 daily，避免外部 API 失败影响生产。

## Findings From 2026-05-11 Work
- Security 中文源低不是单纯缺源：关闭跨日 raw 去重后，72h 模拟候选从 49 条变为 108 条，中文候选约 30 条。
- 之前 `source_profile` 会把生成后的中文摘要算作中文源，导致报告误判 AI/finance 几乎全中文；已修复为只看原始标题、源域名和 registry。
- AIHOT 对比显示我们的 AI 板块更偏“工程/模型/安全”，AIHOT 更偏“选题/产品/投资/X 热点”；后续要靠 source registry 和 final_score 决定是否吸收这类信号，而不是把 AIHOT 作为主源。
