# Week 2026-05-25 Optimization Plan and Execution Notes

## Starting Point

The 2026-05-19 to 2026-05-24 window showed clear progress on the traditional security board but exposed three issues:

- `daily-digest` for 2026-05-25 was cancelled after 45 minutes while stuck in the `Digest security` Gemini step.
- `ai_security` was too strict after the previous cleanup: average `6.7/10`, with only `2/6` full days.
- `ai` was not stable enough: average `14.2/15`, Chinese source target only `1/6` days, and Google News could exceed the intended cap on sparse days because cap relaxation relaxed aggregate caps.

## Changes Made

### P0 Reliability

- Added `GEMINI_REQUEST_TIMEOUT_SEC` support to the Gemini backend.
- Wrapped Gemini SDK calls in a thread-level timeout so a single request cannot hang the whole workflow.
- Increased workflow job timeout from 45 to 70 minutes.
- Added per-board shell timeouts in GitHub Actions:
  - fetch: 8 minutes
  - digest: 15 minutes
- Workflow now continues to later boards if one board fails, builds available artifacts, then fails at the end if any board failed.

### AI Security

- `ai_security.fill_score_floor` changed from `5` to `4`.
- Generic AI news remains capped at score `3`, so this recovers borderline technical security items without reopening泛 AI product/news filler.
- Added direct AI security source: HiddenLayer Blog.
- Added a focused Google News fallback query for AI/code/agent supply-chain security.
- Added board-specific `google_news` kind penalty for `ai_security`.

### AI Frontier

- `ai.fetch_hours` changed from `30` to `48` to reduce sparse-day underfilling.
- Added `relax_aggregate_caps: false` so cap relaxation can fill with direct/source diversity but cannot exceed Google News / aggregator caps.
- Added board-specific scoring overrides:
  - AI Google News penalty strengthened to `-1.6`.
  - official/expert X signals get a small AI-specific boost.

### Source Registry

- Registered selected unknowns and important sources:
  - `blogs.nvidia.com`, `blog.google`
  - `brutecat.com`, `guage.cool`, `krebsonsecurity.com`
  - `hiddenlayer.com`, `lakera.ai`, `portswigger.net`, `embracethered.com`
  - X handles: `LakeraAI`, `HiddenLayerSec`, `wunderwuzzi23`

A local source-profile recomputation over recent digest files now returns zero unknowns under the updated registry; generated docs will reflect this after the next `site_builder.py` run.
